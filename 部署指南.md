# 附录：系统启动与部署指南

## 1. 环境要求

- Node.js **18.11 或更高版本**（`npm run dev` 依赖 `node --watch`，18.10 及以下不支持）
- SQL Server 2019 Express 或更高版本（安装时选择**混合认证模式**）
- SQL Server Management Studio（SSMS）

```bash
node --version   # 验证：应显示 v18.11.x 或更高，低于此版本先升级 Node.js
```

## 2. 数据库初始化

### 2.1 启用 TCP/IP 协议并放行防火墙

SQL Server 全新安装后 TCP/IP 协议默认**禁用**，且 Windows 防火墙默认拦截 1433 端口，两项均需处理。

**自动方式**：右键 `enable-sql-tcp.bat` → 以管理员身份运行。运行后继续按下面 2.1.1 检查是否真正生效。

**手动方式**（bat 脚本不生效时使用，不同 SQL Server 版本的注册表路径不同会导致脚本失效）：

1. 打开 **SQL Server Configuration Manager**（开始菜单搜索）
2. 展开 **SQL Server 网络配置** → 点击实例的协议（默认实例：`MSSQLSERVER 的协议`；Express：`SQLEXPRESS 的协议`）
3. 右侧 **TCP/IP** → 右键启用 → 双击打开 → **IP 地址** 选项卡
4. 拉到最下方 **IPAll** → **TCP 端口** 填入 `1433` → **TCP 动态端口** 清空 → 确定
5. 打开 **SQL Server 服务** → 右键 SQL Server 实例 → 重新启动

#### 2.1.1 验证 TCP/IP 与防火墙

```bash
# PowerShell 中执行：确认 1433 端口处于 LISTENING 状态
netstat -an | findstr ":1433"
```

如果无输出，检查：
- **SQL Server Browser** 服务是否启动（Express 命名实例必须）：`services.msc` → 找到 `SQL Server Browser` → 启动类型改为「自动」→ 启动
- **Windows 防火墙**：控制面板 → Windows Defender 防火墙 → 高级设置 → 入站规则 → 新建规则 → 端口 → TCP、1433 → 允许连接

## 3. 后端启动

1. 进入 `sts-dams-backend`，将 `.env.example` 复制为 `.env`
2. 编辑 `.env`，按实际环境修改：

   | 变量 | 怎么填 |
   |------|--------|
   | `DB_SERVER` | 默认实例 → `localhost`；Express → `localhost\SQLEXPRESS` |
   | `DB_PASSWORD` | 创建 `dams_user` 时设的密码 |

3. 终端执行：
   ```bash
   npm install
   npm run dev
   ```
4. 验证：浏览器打开 `http://localhost:8080/api/health`，`database` 为 `"connected"` 即正常

**常见报错速查：**

| 报错信息 | 原因 | 解决 |
|----------|------|------|
| `ECONNREFUSED 127.0.0.1:1433` | TCP/IP 未启用或防火墙拦截 | 回到 2.1 节逐步排查 |
| `port 1433 not valid / not listening` | SQL Server 用了动态端口而非 1433 | 回到 2.1 手动方式，IPAll 中清空动态端口、填入 1433 |
| `Login failed for user 'dams_user'` | 密码错误或用户未创建 | 在 SSMS 中重跑 2.2 节的 CREATE LOGIN/USER |
| `ECONNREFUSED` 但 `localhost` 可连 SSMS | 实例名带了 `\SQLEXPRESS` 后缀 | `.env` 中 `DB_SERVER` 加上 `\SQLEXPRESS` |
| `EADDRINUSE :::8080` | 8080 端口被其他程序占用 | `.env` 中 `PORT=` 改为 `3001`；同时修改前端 `vite.config.ts` 中 `proxy.target` 为 `http://localhost:3001` |

## 4. 前端启动

1. 进入 `sts-dams-frontend`，终端执行：
   ```bash
   npm install
   npm run dev
   ```
2. 浏览器打开 `http://localhost:3000`，出现登录页即成功

> **先启动后端再打开前端**。后端未运行则页面空白，非故障。
>
> 如果 `npm install` 速度很慢，先执行：`npm config set registry https://registry.npmmirror.com`
>
> 如果 3000 端口被占用：Vite 会自动尝试下一个可用端口（3001、3002…），留意终端输出中的实际地址。
